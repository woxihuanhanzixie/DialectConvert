# **《声临其境》项目 Demo 一期技术实现文档：普通话识别与方言语义生成**

## **一、 阶段目标与核心场景** 本阶段 Demo 主要针对“Z世代‘社交货币化’场景”，打通项目主链路的前半部分。主要目标是实现：学生通过前端录制普通话语音，系统能够准确识别并利用 AI 技术将其转化为带有特定方言俚语与深层语义的方言文本表现形式。

## **二、 核心技术模块设计**
### gold-teacher路线：
**模块 1：语音输入与普通话识别 (Mandarin ASR)**

- **功能描述**：接收用户在前端（如微信小程序）录制的普通话语音，将其精准转化为普通话文本流。
- **技术实现细节**：
    - **前端处理**：结合基础声学特征处理技术，优化并提升在噪声环境（如校园、宿舍）下的语音切分效果，确保输入音频的质量。
    - **识别引擎**：本阶段可优先通过 RESTful API 调用成熟的商业语音识别接口（如讯飞语音识别接口）来保证普通话转写的准确率与低延迟响应。
    - **框架与部署**：通过 Python 编写后端逻辑进行接口对接，采用端云协同架构保障响应速度。

**模块 2：语义转化与方言文本生成 (LLM + RAG)**

- **功能描述**：将识别出的普通话文本转化为符合目标方言习惯的文本，重点是对濒危词汇和地方俚语的精准映射与替换。
- **技术实现细节**：
    - **LLM 智能中枢**：采用大语言模型（LLM）作为控制中枢，构建智能代理（Agent）工作流。LLM 将利用其上下文学习能力动态调度底层的检索模块与生成任务。
    - **RAG 检索增强引擎**：引入检索增强生成（RAG）技术，这是本项目解决传统商业工具“缺乏语义理解、难以处理俚语”痛点的核心差异化优势。
    - **知识图谱与向量索引**：基于团队前期构建的“濒危词汇知识图谱”与“语义向量索引”。当 LLM 接收到普通话文本后，提取核心语义并向 RAG 引擎发起检索，通过语义向量映射技术，匹配出对应的方言特色词汇，有效缓解方言中“有音无字”的转写难题。
    - **自适应调整**：利用大模型 API，系统不仅能生成方言文本，还能根据用户设定的水平或场景（如“方言表情包”等轻量传播形式）自适应调整输出方言的深度和趣味性。

**三、 阶段性系统架构与工作流**

1. **数据层**：依托已整合的公开方言库及团队收集的多地域小规模高质量平行语料，建立普通话-方言平行样本和濒危词汇标签体系。
2. **交互层**：用户在微信小程序端发起语音请求，需确保主要交互响应时间控制在 500ms 以内。
3. **云端处理流**：
    - 小程序通过 RESTful API 将音频传至云服务器。
    - 云端调用 ASR 接口完成“音频 $\rightarrow$ 普通话文本”。
    - 普通话文本输入至 LLM Agent 工作流。
    - Agent 触发 RAG 模块，在方言语义向量索引中进行比对与知识获取。
    - LLM 整合检索结果，生成最终的方言文本与标注要素返回给前端。

**四、 开发分工与算力准备**

- **算力支持**：本阶段的前端开发、数据预处理和接口联调可由团队成员的个人电脑承担；而涉及 RAG 向量索引构建等计算密集型任务，则需利用云端 GPU 资源与学生优惠算力进行支持。
- **后续扩展接口预留**：本阶段生成的“方言文本与要素特征”将作为结构化数据，直接为后续的 TTS（零样本个性化音色克隆）与数字人驱动模块提供输入依据。

### 《声临其境》项目 Demo 二期技术演进文档：方言双向闭环与多场景拓展

#### 一、 核心攻关：功能 6 —— 方言与濒危词汇识别转写（逆向链路打通）

Demo1 实现了“普通话识别与方言生成”，而功能 6 的核心是打通**“用户输入方言语音 $\rightarrow$ 系统识别并转写为普通话”**的逆向链路。这是技术难度最高的部分，因为方言普遍面临数据匮乏的问题。

**1. 复杂环境下的声学前端处理**

- **技术要点**：由于方言语料多来自真实生活场景（如长辈录音、乡村导览），前端必须结合基础声学特征处理技术，有效滤除噪声，提升在复杂环境下的语音切分与端点检测效果，保障后续识别的有效输入。

**2. 基于小样本学习的低资源方言 ASR 建模**

- **技术要点**：针对方言平行语料严重不足的痛点，不能仅依赖商业 API。团队需以 **TensorFlow/Keras** 为核心框架，引入**小样本学习（Few-shot Learning）算法**进行深度学习与迁移学习。
- **技术指标**：通过在开源预训练底座（如基于 Transformer 的架构）上进行微调，目标是使特定语种条件下的方言识别准确率达到 **85% 以上**，远超一般通用模型低于 60% 的行业基准。

**3. RAG 反向映射：攻克“有音无字”难题**

- **技术要点**：方言中存在大量“有音无字”的濒危词汇和地方俚语，传统的语音识别往往会输出乱码或同音错字。
- **系统升级**：复用并升级 Demo1 的 RAG 知识引擎与濒危词汇知识图谱。基于词频统计和语义分析，当 ASR 输出拼音序列或模糊文本时，系统在语义向量索引中进行比对与检索，通过**语义向量映射技术**，将方言词汇精准转写为符合上下文的普通话表意文本，完成深层语义的解码。

---

#### 二、 场景拓展：功能 3 —— 适老化“乡音陪伴”与怀旧治疗

功能 3 是 Demo1 中 TTS 技术与 LLM 角色扮演能力在银发经济场景（特别是辅助改善阿尔兹海默症情绪）下的垂直深化。

**1. 零样本/小样本音色克隆（复用与定制 TTS）**

- **技术要点**：复用 Demo1 验证过的零样本音色克隆技术（TTS），但目标对象发生改变。系统需提取远方子女的少量语音样本，克隆出子女的声音特征，同时保留方言特有的语气助词与变调，合成极具真实感与亲切感的方言对话。

**2. 适老化 LLM Agent 对话中枢**

- **技术要点**：调整大模型（LLM）的 Prompt 工程。结合方言知识图谱，向 LLM 注入特定地域的方言童谣、谚语和俚语数据，使其具备“怀旧治疗”的对话策略，从而建立温暖的数字纽带，缓解老年人孤独感。

---

#### 三、 数据飞轮与平台化：功能 4 —— 众包采集与“全球乡音地图”

功能 4 的实现不仅是为了服务海外华侨的“数字寻根”，更是为上述所有 AI 模型（特别是功能 6 的 ASR 模型）提供源源不断的活态数据训练集。

**1. “方言树”众包采集架构开发**

- **技术要点**：在微信小程序前端开发“方言树”与“全球乡音地图”功能模块。建立一套轻量化、易操作的用户录音与上传机制，鼓励年轻用户邀请长辈共建，收集碎片化、活态化的方言语料。

**2. 数据清洗、标注与自动化增强链路**

- **技术要点**：众包收集到的数据包含大量脏数据。后端需要建立自动化的数据处理管线：通过清洗、小样本发音标注规范以及数据增强技术（如加噪、变速等），扩充平行样本库，进而反哺和提高功能 6 中方言模型的鲁棒性。

### 总结：从 Demo1 到全功能的系统架构演进

在加入功能 3、4、6 后，本项目的技术架构将正式闭环：

- **数据层**：通过**功能 4（众包采集）**形成数据飞轮，不断丰富方言/普通话平行语料库和知识图谱。
- **核心引擎层**：左手依赖**功能 6（方言 ASR + RAG）**实现“听懂方言并理解深意”，右手依赖 **Demo1/功能 3（定制 TTS + 知识生成）**实现“说出地道方言与个性化声音”。
- **应用层**：根据用户特征自适应分发场景——对 Z 世代提供表情包与配音（Demo1），对银发族提供陪伴（功能 3），对海外华人提供寻根地图（功能 4）。
