# RAG 方言语义增强 — 实现方案

> 状态：骨架已完成 · 知识库已扩充演示高频语义 · 待母语者审校
> 更新：2026-06-15

## 定位

RAG 模块不替代 Qwen LLM API，而是在**方言改写之前**注入方言词汇/俗语/习惯表达，让 LLM 输出更地道的方言文本。

```
ASR → 情绪分析 → RAG检索方言知识 → 方言改写(prompt注入) → TTS
```

## 架构

```
app/rag/
├── __init__.py          # 模块入口
├── knowledge_base.py    # JSON 知识库加载、缓存、关键词检索
├── retriever.py         # jieba 分词 + 关键词匹配检索器
└── data/
    ├── cantonese.json   # 粤语词汇对照表 (70 条目)
    ├── sichuanese.json  # 四川话词汇对照表 (70 条目)
    └── hokkien.json     # 闽南话词汇对照表 (73 条目)
```

## 检索流程

1. **分词**：`jieba.cut()` 对 ASR 原文进行中文分词；jieba 未安装时退化为字符二元组
2. **匹配**：每个 token 与知识库 `keyword` 字段做双向子串匹配
3. **排序**：按匹配 token 数计分，返回 Top-5
4. **注入**：格式化为 prompt snippet，插入 `rewrite_to_dialect()` 的 LLM 调用

## 知识条目格式

```json
{
  "keyword": "年轻人",
  "dialect_expression": "后生仔 / 后生女",
  "category": "人称",
  "usage_note": "口语常用，男称仔女称女",
  "context": "泛指年轻一代"
}
```

## 效果示例（粤语）

**无 RAG**：
> 我想把呢句话变成家乡话，等更多年轻人听到方言嘅味道。

**有 RAG**（检索到「变成→变做」「年轻人→后生仔」「味道→韵味」「听见→听到」）：
> 我想将呢句说话变做家乡话，等更多后生仔听到方言嘅韵味。

### 短期（比赛前）
- 每种方言扩充到 80-100 条目
- 补充情景口语模板（问路、问候、道别、讲故事等）
- 补充情感相关词汇（开心、难过、惊讶等）

### 中期（比赛后）
- 切换到向量检索（sentence-transformers 本地 embedding + faiss）
- 引入繁体中文白读/文读标注
- 接入方言 CID/ISO 639-3 编码体系

### 长期
- 引入方言知识图谱（见下文）
- 构建濒危方言语义关系网（近义、反义、上下位）
- 支持用户贡献和纠错

## 方言知识图谱 — 后续计划

### 目标
从"词 ↔ 词对照"升级到"概念 → 关系 → 语境"的结构化图谱，让改写不只是替换单个词，而是理解语义关系后重述。

### 图谱 Schema 示例
```
[年轻人] --is_equivalent_to--> [后生仔]  (粤语)
[后生仔] --has_hypernym--> [人/人士]
[后生仔] --has_connotation--> [亲切、接地气]
[后生仔] --used_in_context--> [日常交谈、市井叙述]
[后生仔] --contrasts_with--> [老人家/长辈]
```

### 候选工具
| 工具 | 说明 |
|------|------|
| Neo4j | 图数据库，适合存储和查询方言语义关系 |
| NetworkX | Python 图计算库，轻量原型首选 |
| Apache Jena | RDF 三元组知识图谱框架 |
| Protégé | 本体编辑器，可定义方言领域本体 |

### 数据采集渠道
- 方言田野调查录音转写
- 方言电影/电视剧字幕（如港剧、川渝影视）
- 社交媒体方言语料（抖音/B站方言视频评论）
- 地方志/民俗文献中的方言记载

## 注意事项

1. **RAG 是辅助而非替代**：LLM 仍然负责核心的语义理解和句子生成；RAG 仅提供词汇级别的正确参考。
2. **知识库需要方言母语者审校**：JSON 条目目前由 LLM 辅助生成，真人审校后标注 `reviewed: true`。
3. **检索噪音控制**：当前 `top_k=5`，过多匹配项可能引入不相关词汇；后续可通过 Jaccard 相似度阈值过滤。
4. **不改变 Prompt 结构**：RAG 上下文以独立段落注入，与情绪分析、方言 instruction 解耦，便于单独调试和关闭。
