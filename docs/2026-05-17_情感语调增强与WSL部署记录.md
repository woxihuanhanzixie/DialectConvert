# 2026-05-17 情感语调增强与 WSL 部署记录

## 背景

当前公网链路已经可以完成 ASR、方言改写、音色复刻和 CosyVoice 方言合成，但用户反馈最终语音仍偏平，人物情绪和说话腔调不够接近输入音频。PowerShell 在部署时也多次暴露中文路径、ZIP 压缩、远端 Linux 解包和 Git 网络处理问题，因此后续部署切换为 WSL/Ubuntu 优先。

## 本次改动

- 新增 `analyze_expression`：对 ASR 文本恢复标点，生成 `emotion_label` 和短 `prosody_instruction`。
- `source_text` 改为前端展示带标点版本，让用户能看到人物情绪和停顿。
- 方言改写提示接收情绪与语调信息，要求保留原句标点承载的停顿、惊讶、焦急、开心或低落。
- 新增 `build_tts_instruction`：把官方方言指令与短情绪提示合并，控制在 CosyVoice 适合的短 instruction 内。
- 前端结果区新增“情绪语调”展示，不改变录音、上传、提交和播放逻辑。
- 更新测试，确保合成文本仍使用方言文本，instruction 同时包含方言与情绪语调。

## 链路变化

旧链路：

```text
音频 -> ASR -> 方言改写 -> 音色注册 -> 方言 TTS
```

新链路：

```text
音频 -> ASR -> 标点/情感/语调分析 -> 方言改写 -> 短 TTS 指令 -> 音色注册 -> 方言情感 TTS
```

## 边界说明

CosyVoice 的 `instruction` 可以控制方言、情感或角色，但它不是完整的语音风格迁移模型。当前实现能把“标点、情绪标签、语调描述”传递到 TTS，提升自然度和情绪起伏；如果要达到严格复制输入音频的每个停顿、音高曲线和情感强度，需要额外的韵律分析/语音风格迁移能力或更强的音频到音频模型。

最低保障仍然是：不牺牲原本可用的方言输出和音色复刻链路。

## 部署方式

后续优先使用 WSL/Ubuntu：

```bash
cd /mnt/d/dialect\ convert
bash scripts/deploy_tencent_cloud_tar.sh 43.139.53.84 root /opt/dialect-convert 7860 http://43.139.53.84
```

部署后通过 `/health`、`systemctl is-active dialect-convert` 和一次真实 `/api/convert` 请求验收。
