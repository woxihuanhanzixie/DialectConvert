# 2026-05-17 前端体验与运行时数据治理记录

## 本次目标

把归档重建版 Web 应用从“表单上传页”升级为可现场演示的移动端录音体验，同时不改变后端主链路：

```text
录音/上传 -> ASR -> 方言改写 -> 音色注册/复用 -> Gold Teacher + Voice Matched 合成
```

## 前端实现

- 重做 `static/index.html`、`static/styles.css`、`static/app.js`。
- 首屏改为手机优先的 Live 录音台，参考移动端实时输入产品形态：
  - 顶部显示 Live 状态。
  - 中央大面积实时波形和计时器。
  - 底部提供精简加号上传、录音、停止三个主控件。
  - 方言选择固定为粤语、四川话、闽南话三项。
- 上传本地音频时使用隐藏文件输入和加号按钮，不把默认浏览器文件框暴露给用户。
- 生成阶段展示四步进度：接收音频、识别原文、改写方言、复刻音色。
- 结果区明确区分：
  - Voice Matched 克隆音色，成功时作为推荐播放。
  - Gold Teacher 标准方言，作为稳定参考或回退结果。
  - 原始识别文本、方言表达、发音提示和警告。

## 运行时数据治理

- `new_job_id()` 改为 `dc_YYYYMMDDHHMMSS_xxxxxxxx`，方便按时间排查与清理，不包含用户原始文件名。
- 上传仍保存到 `runtime_data/uploads/{job_id}.{ext}`，输出仍保存到 `runtime_data/outputs/{job_id}_{label}.mp3`。
- 新增 `runtime_data/jobs/{job_id}.json` 任务元数据，只记录扩展名、MIME 类型和字节数，不记录原始文件名和音频内容。
- `cleanup_runtime()` 现在统一清理：
  - `runtime_data/uploads`
  - `runtime_data/outputs`
  - `runtime_data/jobs`
  - 过期的 `runtime_data/voice_cache/*.json`
- 普通运行时文件由 `CLEANUP_AFTER_HOURS` 控制，音色缓存由 `VOICE_CACHE_TTL_HOURS` 控制，默认 720 小时。

## 部署注意

- 腾讯云 50G/双核机器继续使用轻量 FastAPI + systemd + Nginx 反代，不在服务器部署大模型。
- 生产 `.env` 必须配置 `PUBLIC_BASE_URL=http://43.139.53.84` 或正式域名，否则云端 ASR/音色注册无法回拉上传音频。
- 不提交 `.env`、私钥、运行时音频、缓存目录。
- 建议保持 `CLEANUP_AFTER_HOURS=24`，比赛演示场景足够复盘，同时不会堆积大量临时音频。
