# 2026-05-18 短音频校验与录音体验修复记录

## 背景

真实手机测试中，2-3 秒参考音频会进入后端主链路，随后在 CosyVoice 音色复刻阶段返回 `Audio.AudioShortError` 或 `audio too short`。前端还存在两个体验问题：手机系统录音回到页面后显示的时长可能是离开网页的时间，iOS 在 HTTP 页面使用 `capture="microphone"` 时可能跳到视频录制入口，上传后的音频也缺少就地删除入口。

## 本次修复

- 后端新增 `app/audio_utils.py`，优先使用 `ffprobe` 读取音频时长，WAV 使用 Python 标准库兜底。
- `/api/convert` 保存上传文件后先校验参考音频时长，不满足 `SPEAKER_REF_AUDIO_MIN_S` 时直接返回中文错误：`请输入大于 Ns 的音频`。
- 新增 `/api/audio-limits`，让前端按服务端真实配置读取最短/最长参考音频限制，避免前后端提示不一致。
- 将 CosyVoice 的 `Audio.AudioShortError`、`audio too short` 等外部错误转译为中文警告，不再把英文 JSON 原样展示给用户。
- 前端上传/录制后通过 `<audio preload="metadata">` 读取真实音频时长，并显示在文件信息中。
- 网页内录音从小于 0.8 秒才拦截，改为按后端配置的最短时长拦截。
- 上传预览卡片新增删除按钮，可清空音频后重新录制或上传。
- iOS 非安全上下文不再强制使用 `capture="microphone"`，避免被系统带到视频录制；HTTPS/安全上下文仍优先使用网页内录音。

## 部署注意

生产 `.env.prod.example` 当前建议 `SPEAKER_REF_AUDIO_MIN_S=10`、`SPEAKER_REF_AUDIO_MAX_S=20`。腾讯云部署仍使用 WSL/Linux 下的 `scripts/deploy_tencent_cloud_tar.sh`，部署后用 `/health`、`/api/audio-limits` 和一次真实录音转换检查链路。
