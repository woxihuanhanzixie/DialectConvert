# 《声临其境》路演版汇报纲领

适用场景：2026 年度暨南大学 “AI+” 应用创新大赛初赛，团队汇报不超过 4 分钟。

核心调整：不要把 PPT 做成技术说明书，也不要堆模型名。路演要让评委快速听懂三件事：我们解决什么问题、我们已经做出什么可演示闭环、它为什么有价值和创新性。

## 一、汇报主线

一句话定位：

《声临其境》是一个 AI 方言语音体验平台，让用户用普通话、文字或录音输入，快速生成粤语、四川话、闽南语等方言语音，把方言保护从“静态存档”变成“可听、可玩、可分享”的活态体验。

建议主叙事：

```text
方言正在离开年轻人的日常
-> 传统记录方式难以形成互动和传播
-> 我们用 AI 做成可操作的方言语音工作台
-> 当前已打通文字/语音输入到方言语音输出的 Demo 闭环
-> 未来沉淀方言知识库、乡音地图、数字人陪伴等文化应用
```

## 二、4 分钟汇报页纲

### 1. 封面：声临其境

建议时长：15 秒。

页面重点：

- 项目名：《声临其境》
- 副标题：AI 赋能的方言语音生成与传承平台
- 一句话标语：让家乡话被重新听见

讲法：

各位老师好，我们的项目是《声临其境》。我们想解决的问题不是简单把普通话翻译成方言文字，而是让用户真正听到一段自然的方言声音，让方言重新进入年轻人的表达和传播场景。

### 2. 痛点与价值：方言保护缺少年轻人的入口

建议时长：35 秒。

页面重点：

- 痛点 1：年轻人听得懂但不会说、不敢说、不常说。
- 痛点 2：传统保护多是录音、词典、档案，价值高但互动弱。
- 痛点 3：方言学习和传播缺少低门槛、可分享的数字化入口。

价值表达：

我们的价值不是替代专家采集和学术保护，而是补上“公众参与入口”。让用户能用手机录音、输入一句话，立即听到方言表达，愿意玩、愿意分享，方言才有机会从资料库回到生活里。

### 3. 建设方案：AI 方言语音体验平台

建议时长：45 秒。

页面重点：

```text
文字输入 / 语音输入
-> ASR 识别
-> 文本轻量清洗
-> 方言风格控制
-> CosyVoice v3-flash 实时合成
-> 浏览器播放方言语音
```

当前已支持：

- 输入方式：文字、上传音频、手机录音。
- 目标方言：粤语、四川话、闽南语。
- 输出方式：实时方言音频播放。
- 稳定机制：实时失败时走非实时兜底，保证现场演示不中断。

讲法：

我们当前 Demo 已经从“能识别”走到了“能听见”。用户说一句普通话，系统先识别出文本，再进行轻量清洗和方言控制，最后调用 CosyVoice v3-flash 合成目标方言语音，并在网页中实时播放。

### 4. Demo 展示：不是宣传页，是可操作工作台

建议时长：45 秒。

页面重点：

- 左侧：文本输入、音频上传、参考音频上传、方言选择。
- 右侧：生成音频、推荐输出、耗时、Trace ID、错误提示。
- 下方：ASR 原文、最终朗读文本、发音指令、RAG/文化百科预留区域。

建议展示顺序：

1. 输入一句普通话或上传一段录音。
2. 选择目标方言，例如粤语/四川话/闽南语。
3. 点击生成，播放输出音频。
4. 简要指出页面会显示耗时和兜底状态，证明系统不是静态样片。

讲法：

这个页面不是放视频截图的展示页，而是一个能现场操作的语音工作台。它支持输入、识别、生成、播放和错误兜底，评委可以直观看到完整链路。

### 5. 技术要点：三层能力，而不是单一 TTS

建议时长：55 秒。

页面重点：

第一层：语音理解

- ASR 把用户录音转成文本。
- 文本清洗处理标点、空格和可读性，保证后续合成稳定。

第二层：方言生成

- CosyVoice v3-flash 负责低延迟方言语音合成。
- 通过方言指令控制粤语、四川话、闽南语等风格。

第三层：演示稳定性与扩展

- WebSocket 实时播放，降低等待感。
- 失败时自动走非实时兜底音频。
- 保留参考音色、RAG 文化百科、发音提示等扩展接口。

讲法：

技术上，我们不是只调一个语音接口，而是把“输入理解、方言生成、现场稳定播放”串成闭环。对路演来说，最重要的是现场能跑、能听、能解释为什么生成这段声音。

### 6. 特色与创新点：从方言翻译到方言体验

建议时长：45 秒。

页面重点：

- 创新 1：结果是声音，不只是方言文字。
- 创新 2：支持多模态输入，降低使用门槛。
- 创新 3：兼顾实时体验和兜底稳定，适合现场展示与后续落地。
- 创新 4：为文化百科、方言知识库、参考音色和数字人联动预留接口。

对比表达：

- 相比传统方言保护：更互动、更适合年轻人传播。
- 相比普通 TTS 工具：围绕方言文化场景做了完整流程。
- 相比单纯文本改写：最终交付的是可听见的语音体验。

讲法：

我们的创新点可以概括为：把方言保护做成一个可体验的 AI 产品。用户不是看一段解释，而是马上能听到、能模仿、能分享，这会让方言传承更接近日常生活。

### 7. 预期成效：短中长期三步走

建议时长：35 秒。

页面重点：

短期：

- 完成比赛 Demo 和路演展示。
- 支持三种方言的语音生成体验。
- 可用于校园方言互动活动。

中期：

- 接入更多方言和表达场景。
- 建设方言文化百科、发音提示和语料沉淀。
- 拓展配音、乡音播报、社交分享玩法。

长期：

- 建设方言树和乡音地图。
- 联动数字人，做适老化乡音陪伴。
- 形成“采集、生成、传播、反馈”的方言数字化闭环。

讲法：

短期我们交付可演示、可体验的 Demo；中期让它进入校园传播和文化活动；长期希望沉淀成方言数字化平台，连接语料、文化解释和用户参与。

### 8. 总结与展望：让方言活在新的媒介里

建议时长：25 秒。

页面重点：

- 方言保护不能只靠记录，也需要新的使用场景。
- AI 可以降低方言表达和传播门槛。
- 《声临其境》的目标是让家乡话被听见、被使用、被传下去。

收束句：

《声临其境》希望用 AI 把方言从档案里的声音，变成手机上能听、能玩、能分享的体验。我们相信，当年轻人愿意重新开口、重新转发、重新使用家乡话，方言保护才真正从记录走向传承。

## 三、PPT 视觉建议

整体风格：

- 少放大段文字，多用流程图、界面截图、关键词。
- 每页只讲一个重点。
- 不做“技术论文型 PPT”，要做“评委 5 秒能看懂”的路演页。

建议画面：

- 第 2 页用“年轻人少说方言”和“传统档案保护”的对比图。
- 第 3 页用一条横向流程链路展示完整方案。
- 第 4 页放真实 Web Demo 截图，标注输入区、方言选择、音频播放区。
- 第 5 页用三层技术架构图：语音理解、方言生成、稳定播放。
- 第 6 页用三列对比：传统保护、普通 TTS、声临其境。
- 第 7 页用路线图：短期 Demo、中期校园传播、长期数字化平台。

## 四、需要避免的表述

不要说成已经完成：

- 数字人已经完整联动。
- 方言树数据库已经建成。
- 全球乡音地图已经上线。
- OpenVoice/RVC 已经是当前公网主链路。
- Gold Teacher 是当前公网主路线。

更稳妥的说法：

- 当前主线是 CosyVoice v3-flash 三方言实时语音 Demo。
- Gold Teacher、OpenVoice、RVC 是早期探索或后续技术储备。
- 数字人、方言树、乡音地图是下一阶段拓展方向。

```text
封面与一句话定位：15 秒
痛点与价值：35 秒
建设方案：45 秒
Demo 展示：45 秒
技术要点：55 秒
创新优势：45 秒
预期成效：35 秒
总结展望：25 秒
```

总计约 4 分钟。现场如果要播放视频，视频控制在 60 到 90 秒以内，并把讲解压缩到“输入、生成、播放、兜底”四个动作。
