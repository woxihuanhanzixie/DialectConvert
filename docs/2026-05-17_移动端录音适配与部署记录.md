# 2026-05-17 移动端录音适配与部署记录

## 背景

手机端录音按钮在部分设备上无法稳定工作，尤其是通过 `http://43.139.53.84` 访问时，移动浏览器会因为页面不是安全上下文而禁止网页直接访问麦克风。原前端还固定使用 `audio/webm`，这会导致 iOS Safari 和部分安卓浏览器生成格式不兼容。

## 本次调整

- 前端录音改为双轨策略：
  - HTTPS/安全上下文且支持 `MediaRecorder` 时，使用网页内实时录音。
  - HTTP、iOS 限制或浏览器不支持时，自动打开手机系统录音器，录完返回后直接把音频作为 `FormData.audio` 提交。
- 网页内录音不再固定 `audio/webm`，改为按 `MediaRecorder.isTypeSupported()` 选择 `audio/mp4`、`audio/webm`、`audio/ogg` 等可用格式。
- 手机交互支持按住录音、松开发送、上滑取消；不支持网页内录音时显示系统录音器状态提示。
- 上传控件允许 `.m4a`、`.mp4`、`.3gp`、`.3gpp`、`.caf`、`.amr` 等手机常见录音格式。
- 后端同步放行移动端常见音频扩展名和 `video/mp4`、`video/quicktime` 这类 iOS 可能返回的媒体类型。
- 修复前端与 `AGENTS.md` 中的 UTF-8 乱码，避免手机端页面提示异常。

## 验收要点

- 安卓 Chrome/微信浏览器：若网页内录音不可用，应自动调起系统录音器；录完返回后页面出现音频预览，并可点击“生成我的方言音色”。
- iOS Safari/微信浏览器：优先走系统录音器或文件选择回退；返回的 `.m4a/.mp4` 应被后端接受。
- 桌面浏览器：HTTPS/localhost 环境可使用网页内实时录音，生成 Blob 后直接提交。
- 后端仍使用原 `/api/convert` 链路，不改变方言改写、音色克隆和 CosyVoice 合成主流程。
