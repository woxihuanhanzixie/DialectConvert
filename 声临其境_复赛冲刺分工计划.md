# 「声临其境」复赛冲刺 · 分工计划

## 总目标
本轮（到下周四复赛）把项目从"方言转换工具"做成、讲成"**方言保护平台**"：
RAG 让方言更地道，应用层让用户共建语料，文书/PPT 把这个故事讲圆，正面回应评委"是不是只能翻译"。

## 大纲：谁做什么
1. **RAG 技术** —— 黎奕阳（林俊健配词库）：让输出从"普通话味直译"变"地道方言"
2. **应用层** —— 范宇恒（林俊健配内容）：做一个能演的"方言知识闯关"
3. **文书撰写** —— 王藜婷：改计划书/报告 + 写答辩话术（详见第三节）
4. **PPT 补齐** —— 王藜婷 + 黎奕阳：补关键几页（详见第四节）

> **本轮原则**：RAG 只做雏形 + 对比演示；应用层做一个能演的 demo；社区/适老/数字人写进"展望"，不强行落地。

## 一、RAG 技术（黎奕阳 / 林俊健）

**目标**：现场能演"开 RAG vs 关 RAG"的明显差异。这一个目标就够，别追求词库做大。

1. **定词条结构**：方言词 / 普通话对应 / 释义 / 例句 / 拼音 / 文化注释 / 濒危等级
2. **建小词库**：每种方言 50–150 条高价值俚语、语气词、"有音无字"濒危词（林俊健主力填；来源：中国语言资源保护工程、汉语方言地图集、众包，文学院图谱若拿到）
3. **向量检索**：embedding + 简单向量库（MVP 用 FAISS 或余弦即可）
4. **接主链路**：在 `rewrite_to_dialect` 检索 top-k 注入 prompt，复用已有的"方言词库预留"接口
5. **加开关 + 备 case**：一个开/关 RAG 的 flag，准备 3–5 个对比 case 的截图/录音

## 二、应用层搭建（范宇恒 / 林俊健）

**做什么**：方言知识闯关（不做全功能社区，社区放展望）。

- 题型先做两种：**看词猜义**（选择题）、**听音辨义**（放方言音频选意思）
- 题目直接从 RAG 词库生成，不另写
- 用户答题或录音后，显示"**已为方言库贡献 N 条语料**"——把"保护"做可见，这是闭环的关键
- 有余力再加：录一句方言打分、校园排行榜

**一句话标准**：能现场演、能体现"用户在共建语料"就够，不求功能全。

## 三、文书撰写（王藜婷）—— 详细

文书 = ① 计划书/报告修订 ② 答辩讲稿。下面是具体写什么。

### 1. 先定一句话核心叙事（全文反复用）
> "声临其境不是把普通话翻译成方言，而是用 RAG 方言知识增强 + 用户共建语料，让濒危方言地道地'活'在年轻人的日常使用里。"

这句话放进：报告摘要、PPT 首页、答辩开场。

### 2. 新增「为什么不只是翻译」一段（回应评委，最重要）
写作抓三个点：
- **翻译是无状态的**：翻一句忘一句，不积累、无壁垒；
- **我们做两件翻译做不到的事**：(a) RAG 注入地道俚语和濒危词，输出不再是普通话味直译；(b) 每次使用都把录音/语料沉淀进方言库；
- **这两件形成飞轮**：用得越多 → 语料越厚 → 方言越地道 → 越多人用。

配一个**具体例子**：某句话关 RAG（普通话直译）vs 开 RAG（地道说法），让对比看得见。

### 3. 技术章节改写
- 把 RAG 从"预留/规划"改成"**已实现的核心模块**"，补：词条结构、检索流程、A/B 对比结果；
- 差异化对比表里"RAG 语义增强"那一行，配一个具体濒危词例子。

### 4. 三处必须统一的口径（不改答辩容易翻车）
- **技术口径**：计划书写的是 TensorFlow 自研训练，实际是 DashScope/Qwen/CosyVoice 等 API + LLM 编排 + RAG。统一改成"**多模型编排 + RAG 增强**"的真实叙事，别写成自研大模型；
- **数字人**：实际没实现，凡提到处标注"规划中"或删掉；
- **指标**：识别率 >85%、MOS ≥4.2、延迟 <500ms——实测过就写实测，没测就写"目标值"。

### 5. 答辩话术（写两段备稿）
- **30 秒开场**：第 1 条那句核心叙事 + 一句现状（MVP 已跑通、支持三种方言）+ 一句价值（每次使用都在抢救濒危方言）；
- **"是不是只是翻译"标准回答**：把第 2 节三个点压成 20 秒，最后甩出 A/B 例子。
