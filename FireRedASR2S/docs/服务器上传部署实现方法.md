# 服务器上传部署实现方法

更新时间：2026-05-14

本文说明如何把本地 `D:\Competition` 工程上传到 Linux 服务器，并部署成可公网访问的 Demo。公钥、私钥、服务器 IP、域名、API Key 均由你自己准备，本文只写实现方法和命令模板。

## 一、部署实现思路

推荐部署形态：

```text
浏览器
  ↓
Nginx 80/443
  ├─ 静态页面：/opt/Competition/FireRedASR2S/public_web
  └─ API 反向代理：/api/* -> 127.0.0.1:8002
        ↓
    dialect_service FastAPI
        ↓
    ASR / 文本审查 / 方言改写 / Qwen-DashScope TTS / Voice Copy
```

也就是说，服务器上不直接暴露 Python 服务端口，只让 Nginx 对外开放 `80` 和 `443`。后端 `dialect_service` 只监听 `127.0.0.1:8002`，由 Nginx 转发 `/api/`、`/healthz`、`/readyz` 请求。

当前工程已经有部署相关文件：

- `deploy/aliyun/nginx_public.conf`：Nginx 配置模板。
- `deploy/aliyun/dialect_service.service`：systemd 服务模板。
- `deploy/aliyun/env.example.prod`：生产环境变量模板。
- `scripts/start_dialect_service_prod.sh`：后端生产启动脚本。
- `public_web/`：公网轻前端静态页面。

## 二、服务器准备

以下示例以 Ubuntu / Debian 系 Linux 为例。

```bash
sudo apt update
sudo apt install -y git rsync unzip nginx ffmpeg curl
```

安装 Miniconda 或使用服务器已有 Python。建议单独创建环境：

```bash
conda create -n fireredasr2s python=3.11 -y
conda activate fireredasr2s
```

创建运行用户和目录：

```bash
sudo useradd -r -m -s /bin/bash voice-demo || true
sudo mkdir -p /opt/Competition
sudo mkdir -p /etc/voice-demo
sudo chown -R voice-demo:voice-demo /opt/Competition
```

## 三、从本地上传代码

### 方案 A：rsync 上传，推荐

在本地 Windows 上可以使用 Git Bash、WSL 或支持 `rsync` 的终端。假设服务器地址是 `<SERVER_IP>`，登录用户是 `<USER>`：

```bash
rsync -avz --progress \
  --exclude ".git" \
  --exclude ".conda_pkgs" \
  --exclude "openvoice_gpu_venv" \
  --exclude "FireRedASR2S/.gradio" \
  --exclude "FireRedASR2S/runtime_data/web_demo_uploads" \
  --exclude "FireRedASR2S/runtime_data/web_demo_preview" \
  --exclude "FireRedASR2S/runtime_data/step2_output" \
  /d/Competition/ <USER>@<SERVER_IP>:/opt/Competition/
```

如果服务器必须指定私钥：

```bash
rsync -avz --progress -e "ssh -i <你的私钥路径>" \
  /d/Competition/ <USER>@<SERVER_IP>:/opt/Competition/
```

注意：模型权重和运行数据目录可能很大。如果服务器需要本地 ASR、OpenVoice 或 RVC，就要同步对应模型目录；如果只跑公网 API-first 轻量版，可以先不上传大模型和缓存。

### 方案 B：打包后 scp 上传

本地打包：

```powershell
Compress-Archive -Path D:\Competition\* -DestinationPath D:\Competition_deploy.zip -Force
```

上传：

```bash
scp -i <你的私钥路径> D:/Competition_deploy.zip <USER>@<SERVER_IP>:/tmp/
```

服务器解压：

```bash
sudo mkdir -p /opt/Competition
sudo unzip /tmp/Competition_deploy.zip -d /opt/Competition
sudo chown -R voice-demo:voice-demo /opt/Competition
```

如果压缩包解压后多了一层目录，需要确保最终结构是：

```text
/opt/Competition/FireRedASR2S
/opt/Competition/OpenVoiceRuntime
```

## 四、安装 Python 依赖

进入项目：

```bash
cd /opt/Competition/FireRedASR2S
conda activate fireredasr2s
pip install -r requirements.txt
```

如果服务器是 CPU 机器，GPU 版 PyTorch 可能安装较慢或不适配。公网轻量版优先依赖云端 Qwen / DashScope 能力，本地 ASR 和 OpenVoice 可以后续再单独补齐。

## 五、配置生产环境变量

复制模板：

```bash
sudo cp /opt/Competition/FireRedASR2S/deploy/aliyun/env.example.prod /etc/voice-demo/dialect_service.env
sudo nano /etc/voice-demo/dialect_service.env
```

至少需要填写：

```bash
DEEPSEEK_API_KEY=你的 DeepSeek Key
DASHSCOPE_API_KEY=你的 DashScope Key
QWEN_TTS_API_KEY=你的 DashScope 或 Qwen TTS Key
QWEN_LLM_API_KEY=你的 Qwen LLM Key
PUBLIC_APP_ORIGIN=https://你的域名
CORS_ALLOW_ORIGINS=https://你的域名
PYTHON_BIN=/opt/miniconda3/envs/fireredasr2s/bin/python
DIALECT_SERVICE_HOST=127.0.0.1
DIALECT_SERVICE_PORT=8002
```

如果暂时没有域名，也可以先用服务器 IP：

```bash
PUBLIC_APP_ORIGIN=http://<SERVER_IP>
CORS_ALLOW_ORIGINS=http://<SERVER_IP>
```

权限建议：

```bash
sudo chmod 600 /etc/voice-demo/dialect_service.env
sudo chown root:root /etc/voice-demo/dialect_service.env
```

不要把真实 API Key 写回 Git、文档或截图中。

## 六、配置 systemd 后端服务

复制服务文件：

```bash
sudo cp /opt/Competition/FireRedASR2S/deploy/aliyun/dialect_service.service /etc/systemd/system/dialect_service.service
```

确认服务文件中的路径符合服务器实际路径：

```ini
WorkingDirectory=/opt/Competition/FireRedASR2S
EnvironmentFile=/etc/voice-demo/dialect_service.env
ExecStart=/usr/bin/bash /opt/Competition/FireRedASR2S/scripts/start_dialect_service_prod.sh
User=voice-demo
Group=voice-demo
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable dialect_service
sudo systemctl start dialect_service
sudo systemctl status dialect_service
```

查看日志：

```bash
sudo journalctl -u dialect_service -f
```

本机验证：

```bash
curl http://127.0.0.1:8002/healthz
curl http://127.0.0.1:8002/readyz
```

## 七、配置 Nginx 对外访问

复制 Nginx 配置：

```bash
sudo cp /opt/Competition/FireRedASR2S/deploy/aliyun/nginx_public.conf /etc/nginx/conf.d/voice-demo.conf
sudo nano /etc/nginx/conf.d/voice-demo.conf
```

把 `server_name` 改成你的域名或服务器 IP：

```nginx
server {
    listen 80;
    server_name demo.your-domain.com;

    client_max_body_size 80m;

    root /opt/Competition/FireRedASR2S/public_web;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8002;
        proxy_http_version 1.1;
        proxy_buffering off;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location = /healthz {
        proxy_pass http://127.0.0.1:8002/healthz;
    }

    location = /readyz {
        proxy_pass http://127.0.0.1:8002/readyz;
    }
}
```

检查并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

浏览器访问：

```text
http://<SERVER_IP>
http://你的域名
```

## 八、HTTPS 配置

如果需要浏览器录音，正式演示建议配置 HTTPS，因为多数浏览器对麦克风权限要求安全上下文。

使用 certbot 的常见方式：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d demo.your-domain.com
```

证书完成后，把环境变量同步改成 HTTPS：

```bash
PUBLIC_APP_ORIGIN=https://demo.your-domain.com
CORS_ALLOW_ORIGINS=https://demo.your-domain.com
```

重启后端：

```bash
sudo systemctl restart dialect_service
```

## 九、部署后验证流程

按顺序验证：

1. 访问 `/healthz`：

```bash
curl http://127.0.0.1:8002/healthz
curl http://你的域名/healthz
```

2. 打开公网页面，确认页面能加载 `public_web/index.html`、`app.js`、`styles.css`。

3. 用文本输入先跑通，不上传音频，验证 TTS 和后端 API。

4. 上传普通音频，验证 ASR / 文本清洗 / 方言合成链路。

5. 上传 10-20 秒单人声参考音频，验证 Voice Copy 或 fallback 是否清楚返回。

6. 查看服务日志确认无异常：

```bash
sudo journalctl -u dialect_service -n 200 --no-pager
sudo tail -n 100 /var/log/nginx/error.log
```

## 十、常见问题

### 1. 页面能打开，但点击生成失败

先看浏览器 Network 面板，请求是否打到 `/api/v1/dialect/pipeline`。再看：

```bash
sudo journalctl -u dialect_service -f
```

常见原因是 API Key 未填、CORS 域名不匹配、后端服务没启动。

### 2. `/healthz` 本地正常，公网不正常

检查 Nginx：

```bash
sudo nginx -t
sudo systemctl status nginx
```

再确认云服务器安全组是否开放 `80` / `443`。

### 3. 上传音频失败

检查 Nginx 上传大小：

```nginx
client_max_body_size 80m;
```

检查服务器是否有 `ffmpeg`：

```bash
ffmpeg -version
```

### 4. systemd 启动失败

重点检查：

- `PYTHON_BIN` 是否指向真实 Python。
- `/opt/Competition/FireRedASR2S` 是否存在。
- `voice-demo` 用户是否有项目目录读写权限。
- `/etc/voice-demo/dialect_service.env` 是否存在且格式正确。

## 十一、更新部署

本地改完代码后再次同步：

```bash
rsync -avz --progress \
  --exclude ".git" \
  --exclude ".conda_pkgs" \
  --exclude "openvoice_gpu_venv" \
  /d/Competition/ <USER>@<SERVER_IP>:/opt/Competition/
```

服务器重启服务：

```bash
sudo chown -R voice-demo:voice-demo /opt/Competition
sudo systemctl restart dialect_service
sudo systemctl reload nginx
```

验证：

```bash
curl http://127.0.0.1:8002/healthz
```

## 十二、最小可用部署清单

如果只做比赛/演示公网版，最少需要：

- `/opt/Competition/FireRedASR2S/dialect_service`
- `/opt/Competition/FireRedASR2S/fireredasr2s`
- `/opt/Competition/FireRedASR2S/public_web`
- `/opt/Competition/FireRedASR2S/scripts`
- `/opt/Competition/FireRedASR2S/deploy`
- `/opt/Competition/FireRedASR2S/requirements.txt`
- `/etc/voice-demo/dialect_service.env`
- Nginx 配置
- systemd 服务

如果要启用本地 ASR / OpenVoice / RVC，则还需要同步对应模型和运行时目录：

- `/opt/Competition/FireRedASR2S/runtime_data/models`
- `/opt/Competition/OpenVoiceRuntime`
- RVC 模型、index 和独立环境配置

首版建议先跑通 API-first 公网链路，再补本地大模型链路。
